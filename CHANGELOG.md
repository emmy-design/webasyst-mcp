# Changelog

Все значимые изменения в проекте документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
проект следует [Semantic Versioning](https://semver.org/lang/ru/).

## [2.2.0] - 2024-12-11

### Добавлено
- Инструменты валидации UI (`validate_ui_usage`)
- Генерация цветовых схем (`generate_color_scheme`)
- Создание адаптивных лейаутов (`create_responsive_layout`)
- Расширенный набор UI-компонентов (chips, bricks, slider, toggle, skeleton и др.)
- Поддержка плагинов доставки и оплаты (`create_shipping_plugin`, `create_payment_plugin`)

### Улучшено
- Документация UI_COMPONENTS_REFERENCE.md (2000+ строк примеров)
- Шаблоны создаваемых приложений теперь включают локализацию по умолчанию
- Улучшена структура создаваемых плагинов

## [2.1.0] - 2024-12-10

### Добавлено
- Инструменты для работы с локализацией (`generate_po_template`, `compile_mo`)
- Проверка соответствия проекта (`check_project_compliance`)
- Сборка релизного архива (`prepare_release_bundle`)
- Документация LOCALIZATION_GUIDE.md

### Улучшено
- Создание BackendLoc action при генерации приложений
- Шаблоны с поддержкой JS-локализации

## [2.0.0] - 2024-12-09

### Добавлено
- Полная поддержка Webasyst UI 2.0
- Инструменты для Site: плагины, виджеты, блоки, темы
- Инструменты для Shop: плагины, темы, отчёты
- Создание UI-компонентов (`create_ui_component`)
- Подключение дизайн-системы (`enable_webasyst_ui`)
- SEO-оптимизация и анализ проектов
- DevOps инструменты (nginx, htaccess)

### Изменено
- Переработана архитектура сервера
- Унифицированы все инструменты создания

## [1.0.0] - 2024-12-08

### Добавлено
- Первый релиз
- Базовые инструменты: list_webasyst_apps, get_app_info
- Создание приложений и плагинов
- Работа с темами и виджетами
- CLI интеграция
- Документация README.md и QUICKSTART.md

---

## Типы изменений

- **Добавлено** — новая функциональность
- **Изменено** — изменения в существующей функциональности
- **Устарело** — функциональность, которая будет удалена
- **Удалено** — удалённая функциональность
- **Исправлено** — исправления багов
- **Безопасность** — исправления уязвимостей
